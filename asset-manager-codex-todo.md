Web 核心流程已经基本完整。下一步建议不要继续堆页面，而是做一轮“可长期使用”的加固

1. 优先级最高：
附件闭环
当前只能上传、列出和删除，缺少图片预览及文件下载接口。还应限制文件类型并处理孤立文件。

2. 完整备份与恢复
当前只备份 SQLite，附件不会进入备份，而且只能在服务器生成，无法下载或恢复。建议做成“数据库 + 附件”的压缩包，并增加下载、恢复及旧备份清理。

3. 安全加固
增加登录/注册限流、安全响应头、生产环境关闭 Swagger；移除登录页默认填写的测试账号。若部署到公网，建议进一步调整 Token 存储方式。

4. 账号管理
设置页补充修改昵称、修改密码、退出所有设备；个人部署时可以配置“关闭公开注册”。

第二优先级：
* 给设置页、资产列表及状态操作统一补齐 Loading、失败提示和防重复提交。
* 增加 Web E2E 测试，覆盖注册、登录、创建资产、编辑、归档、附件和导出。
* 增加 CSV 导入、重复资产检测。
* 增加保修到期、维护时间提醒。
* 更新 README：目前仍写着 9 个默认分类，实际已经是 11 个分类和 6 个标签。
